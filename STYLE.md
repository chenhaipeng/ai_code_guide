# 文档规范

> 整理时间：2026-04-19

本文档定义 `ai_code_guide` 工程中所有 Markdown 文档的统一格式。

---

## 文件组织

- 每个工具/主题一个子目录
- 目录名用小写英文，不加编号：`claudecode/`、`harness-engineering/`、`superpower/`
- 文件名用中文，与主题一致：`claude-mem 使用指南与最佳实践.md`

## 文件头

每份文档开头必须包含：

```markdown
# 文档标题

> 整理时间：YYYY-MM-DD
> 信息来源：实测核验 [repo](url) 版本号

---

## 0. ...
```

- 整理时间用绝对日期，不用"今天""近期"
- 信息来源写明核实了哪些仓库/文档

## 标题层级

```
# 文档标题（唯一，H1）
## N. 章节标题（H2，带编号 0-99）
### N.M 小节（H3）
```

- 章节编号从 `0` 开始（0 = 总览/定位）
- 不跳级：H2 → H3 → H4，不出现 H2 → H4

## 表格

- 使用标准 GFM 表格
- 对齐方式：左对齐
- 属性表用 `| 属性 | 值 |` 格式
- 对比表用项目名做行头

## 交叉引用

- 指向本工程其他文档用**相对路径**：`[文字](../harness-engineering/xxx.md)`
- 指向外部用完整 URL：`[文字](https://github.com/...)`
- 每份文档头部和尾部各放一次交叉引用块

## 数据标注

- Stars、版本号等随时间变化的数据，必须标注数据日期
- 用 `截至 YYYY-MM-DD` 或在文档头部统一标注
- 不引用未验证的第三方数据；如无法核实，标注 `⚠️ 未核实`

## 特殊标记

| 标记 | 含义 |
|---|---|
| `⚠️` | 重要警告 / 注意事项 |
| `⭐` | 推荐度标记 |
| `❌` | 反模式 / 不推荐做法 |
| `✅` | 推荐做法 |

## 反模式章节

每份文档应包含"反模式"章节（通常在末尾），列出常见的错误用法。

## 一句话总结

每份文档最后一节应为"一句话总结"，用引用格式：

```markdown
> **一句话总结的内容。** 加粗核心观点，一句话收尾。
```
