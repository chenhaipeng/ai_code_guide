# 跨工具协作指南

> 整理时间：2026-04-19
>
> ⚠️ **本文档定位：多工具搭配的组合推荐与冲突规避**
> 各工具的详细使用见对应专题文档。

---

## 0. 为什么要关心"搭配"

这些工具不是在一个层面上竞争，而是**在不同层面上互补**：

```
┌─────────────────────────────────────────────┐
│  Coding Agent（运行环境）                    │
│  Claude Code / OpenCode / Codex CLI / Aider │
├─────────────────────────────────────────────┤
│  流程纪律（工作流控制）                      │
│  Superpowers / ECC tdd-workflow              │
├─────────────────────────────────────────────┤
│  记忆（上下文持久化）                        │
│  claude-mem / CLAUDE.md                      │
├─────────────────────────────────────────────┤
│  技能（特定场景增强）                        │
│  ECC 单个 skill / anthropics/skills          │
└─────────────────────────────────────────────┘
```

搭配正确 = 每层各一个，互不冲突。
搭配错误 = 同层装两个，互相打架。

---

## 1. 核心原则

1. **每层只选一个主力** — 流程层不装 Superpowers + ECC tdd，记忆层不装 claude-mem + 其他 memory 插件
2. **先少后多** — 先用 1-2 周裸机，确认痛点再加工具
3. **冲突时流程纪律优先** — 流程类 skill 的优先级高于领域类 skill

---

## 2. 推荐组合

### 2.1 组合 A：极简稳定（推荐大多数用户）

```
Claude Code（裸机）
  + CLAUDE.md（手写，< 200 行）
  + anthropics/skills（官方插件）
```

不加任何第三方插件，靠 CLAUDE.md + 内置功能解决 80% 问题。

**何时升级**：用了 2 周，遇到明确痛点。

---

### 2.2 组合 B：纪律 + 记忆（推荐长期项目）

```
Claude Code
  + Superpowers（流程纪律）
  + claude-mem（跨会话记忆）
```

**为什么兼容**：Superpowers 管流程（brainstorm → plan → TDD → review），claude-mem 管记忆（跨会话 context 注入）。两者在不同层面，不冲突。

**注意**：
- Superpowers 的 `compact` skill 与 claude-mem 的自动压缩可能时机重叠，但不会报错
- claude-mem 的 `MEMORY.md` 与 CLAUDE.md 分开管理，不会打架

**详细配置**：
- [Superpowers 使用指南](../superpower/Superpowers 使用最佳实践指南.md)
- [claude-mem 使用指南](../claudecode/claude-mem 使用指南与最佳实践.md)

---

### 2.3 组合 C：ECC 挑选 + 记忆

```
Claude Code
  + ECC 挑选 3-5 个 skill（常驻克隆，按需 cp）
  + claude-mem（跨会话记忆）
```

**为什么兼容**：ECC 的单个 skill（如 security-review、documentation-lookup）不与 claude-mem 冲突。

**注意**：
- **不要**同时装 ECC 的 tdd-workflow + Superpowers
- **不要**装 ECC 的 strategic-compact + claude-mem（两者都管上下文压缩）

**ECC 挑选指南**：[everything-claude-code 深度使用指南](../oh-my-claudecode/everything-claude-code 深度使用指南.md)

---

### 2.4 组合 D：跨模型编排

```
OpenCode + omo
  + AGENTS.md
  + 多模型订阅（Kimi / GLM / GPT / Claude）
```

**适合**：月度 LLM 账单 > $200，或需要用国产模型。

**注意**：omo 本身不含持久记忆，可搭配 claude-mem 的 OpenCode 安装模式。

**详细配置**：[OpenCode + omo 使用指南](../opencode/📚 OpenCode + Oh My OpenCode 使用最佳实践指南.md)

---

### 2.5 组合 E：研究 / 多 harness 实验

```
Claude Code（主力）
  + OpenCode（备用，跑非 Claude 模型）
  + 共享 CLAUDE.md / AGENTS.md
```

**注意**：两个 harness 的 skill/memory 不互通。Claude Code 的 `.claude/` 和 OpenCode 的配置是独立的。

---

## 3. 冲突矩阵

### 3.1 不能同时装的组合

| 组合 | 冲突点 | 后果 |
|---|---|---|
| Superpowers + ECC tdd-workflow | 流程层抢话语权 | 两个 TDD skill 同时触发，流程混乱 |
| Superpowers + ECC verification-loop | 验证流程重叠 | 重复验证，浪费 token |
| claude-mem + ECC strategic-compact | 上下文压缩策略冲突 | 两者在不同时机压缩，可能导致信息丢失 |
| claude-mem + everything-claude-code 全装 | memory/instinct 体系冲突 | ECC 自带 homunculus/instincts 系统，与 claude-mem 的 observation 体系互相干扰 |

### 3.2 可以同时装的组合

| 组合 | 原因 |
|---|---|
| Superpowers + claude-mem | 不同层面（流程 vs 记忆），互补 |
| ECC 单个 skill + claude-mem | 不重叠 |
| ECC 单个 skill + Superpowers | 只要不是同功能的 skill（如 tdd） |
| anthropics/skills + 任何插件 | 官方插件最稳定，不与第三方冲突 |

---

## 4. 按痛点的搭配决策

| 痛点 | 加什么 | 不加什么 |
|---|---|---|
| 跨会话老忘事 | claude-mem | 不要加 ECC 全装 |
| AI 偷懒、跳测试 | Superpowers | 不要加 ECC 的 tdd（二选一） |
| 需要安全审查 skill | ECC security-review | 不要为此全装 ECC |
| 需要 PDF/文档处理 | anthropics/skills | — |
| 模型账单太高 | 换 OpenCode + omo | 不要换 harness 来省 token |
| 想让 AI 自动查最新文档 | ECC documentation-lookup 或 Context7 MCP | — |
| 团队行为不一致 | 团队 CLAUDE.md + Superpowers | 不要靠"每人自由装插件" |

---

## 5. 安装顺序建议

首次配置建议按此顺序，每步用 1 周观察效果：

```
第 1 周：CLAUDE.md + permissions 收纳 + anthropics/skills
         ↓ 确认痛点
第 2 周：加 claude-mem 或 Superpowers（二选一先加）
         ↓ 观察
第 3 周：加另一个（claude-mem 或 Superpowers）
         ↓ 确认还缺什么
第 4 周+：从 ECC 挑 1-2 个领域 skill
```

**核心**：每次只加一个变量，能清晰判断"这个工具有没有用"。

---

## 6. 反模式

### 6.1 ❌ 一天装 5 个插件

→ 无法判断哪个有用、哪个有害。每次只加一个。

### 6.2 ❌ 为了"功能齐全"全装 everything-claude-code

→ 34 skills + 38 agents 同时加载，误触发 + context 膨胀。按需挑选。

### 6.3 ❌ Superpowers + ECC 全装

→ 流程层冲突最严重。如果要同时用，ECC 只装领域 skill（如 security-review），不装流程 skill。

### 6.4 ❌ 忽视 CLAUDE.md 直接加插件

→ CLAUDE.md 的 ROI 远高于任何插件。先把 CLAUDE.md 写好，再考虑插件。

### 6.5 ❌ 多个 harness 同时跑同一个项目

→ 上下文/记忆割裂。主力用一个，另一个只在特定场景用。
